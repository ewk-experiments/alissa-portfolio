Investigation

The existing navigation pattern hid the main menu when users clicked into a details page. VPC is composed of many related microservices that users create back-to-back, meaning they need to constantly navigate between different resource types. The breadcrumb navigation wasn't intuitive for this workflow.

VPC navigation pattern - before state showing hidden main menu

User Feedback

"I keep getting lost trying to find my way back to the resource list. The navigation just disappears when I need it most."

-- Network Engineer

"I don't understand how to get from one service to another. I end up using the browser back button or starting over from the dashboard."

-- Infrastructure Architect

Iterative Design Process

Iteration 1

Initial exploration of persistent navigation concepts. Tested against the existing pattern using balanced comparison methodology.

20% faster on task

Iteration 2

Refined the navigation based on first round of feedback. Improved information hierarchy and visual clarity of the left nav panel.

22% faster on task

Iteration 3

Final design incorporating all research learnings. Persistent left navigation with clear service groupings and contextual breadcrumbs.

60% faster on task

Stakeholder Alignment

Presented findings to the design system team, product managers, and UI developers. The research data from balanced comparison testing provided the evidence needed to get this design-led initiative prioritized on the roadmap.

Presentation to stakeholders showing research results and design recommendations

Implementation

The new navigation launched in December 2020. Post-launch analytics over 5 months confirmed the impact: a 37% increase in left nav usage and a 45% decrease in breadcrumb usage, validating that users were finding the persistent navigation more intuitive.

"How did you get this design-led initiative on the roadmap?"

-- Fellow designer, asking about the approach

The key was building a strong evidence base through iterative research. Each round of balanced comparison testing added more data to the case, making it easier for stakeholders to prioritize the work alongside feature development.

Summary

Built a Chrome extension that scrapes data from any job listing and saves it to job tracking spreadsheet in Airtable with one click - speeding up workflow by 28x per listing. Started with curiosity about AI-assisted development and evolved into a working tool used in actual job search.

28x faster
96% reduction in time per listing (86 seconds manually vs 3 seconds)

Problem

Staying organized and tracking job applications is essential. Found herself wasting time manually copying information from job postings into tracking spreadsheets - a repetitive, error-prone process that added up quickly across dozens of applications.

The challenge: How to automate copying job details into Airtable spreadsheet?

Solution

One-click Chrome extension that automatically extracts key data from any job posting and uploads to Airtable. Setup requires Airtable PAT, base ID, and table name. Used VS Code with Claude Code extension (Opus 4.5 model).

Extension popup UI and Airtable integration

Process

Phase 1: Experimentation

Started by experimenting with the free version of Claude, learning Chrome extension architecture from scratch. Initial requirements were too narrow, which limited the scope of what the tool could do.

Phase 2: MVP

Started fresh with v0 for brainstorming and ideation, then moved to Claude Code for real iteration and development. This shift from exploration to focused building was where the project gained momentum.

Key decisions she directed

  • Field selection - choosing which job posting data to extract
  • Scope - deciding on a Chrome extension as the delivery format
  • Markdown formatting for job description content
  • Domain-based company detection logic

Outcomes

96% reduction
86 seconds manually vs 3 seconds with the extension

Works across major job platforms: Greenhouse, Workday, Lever, Ashby, and LinkedIn.

Next Steps
